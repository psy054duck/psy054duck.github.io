41. First Missing Positive

Description

Given an unsorted integer array, find the first missing positive integer.

For example,
Given [1,2,0] return 3,
and [3,4,-1,1] return 2.

Your algorithm should run in $O(n)$ time and uses constant space.

Solution 1

此题的描述很简单，就是在一个数组当中找到第一个缺失的正整数。

最直观的解法就是给这个数组排序，随后从第一个正整数1所在位置开始遍历数组，直到遇到一个整数与后一个整数相差不为1时，那么前一个整数加1就是目标。这个解法无法同时达到题目要求的时间与空间复杂度，所以下面只分析复杂度，不上代码。

首先在排序算法的选择上有两个方向，如果选择了基于比较的排序，那么时间复杂度最优为O(nlogn)，但是空间复杂度有可能达到O(1)(例如堆排序）。如果排序算法选择了非比较排序（例如计数排序），那么时间复杂度为O(n)，但是空间复杂度就需要O(n)。

Solution 2

基于上述考虑，我打算修改一下计数排序的实现方式，使得它保留O(n)的时间复杂度，同时还能达到O(1)的空间复杂度。通常计数排序需要一个额外的数组负责计数，但是在本题可以将输入数组作为计数数组，只需改变一下计数方式。

用sz代表输入的数字个数，那么缺失的正整数一定在[1, sz]当中。因为如果缺失的正整数大于sz，那么[1, sz]应该也在输入的数字当中，此时这个集合内的整数个数至少为sz+1，与前提不符。

于是我们就可以通过计数排序的方式找出第一个计数为0的正整数，这个数即为所求。与常用的计数排序不同的是，在计数的时候我们不是+1，而是+maxNum+1，其中maxNum代表这些数字中的最大值。于是数组中的值整除maxNum+1所得结果便是相应数字出现的次数，而数组中的数字对maxNum+1取模，所得结果即为原始输入时相应位置的数字。由于目标数字一定不是负数，所以我们将负数转换为0，下面以[1, -1, 2, 4,]为例。

index 0 1 2 3
nums 1 -1 2 4

将负数归零

index 0 1 2 3
nums 1 0 2 4

计数后(maxNum为4)

index 0 1 2 3
nums 6 5 7 4

此时数组整除maxNum+1

index 0 1 2 3
nums 1 1 1 0

注意，数组中的nums[0]表示非正数的个数。从数组中可以看到1和2均出现了一次，而3没有出现，所以缺失的正整数为3。因为不需要使用额外的数组，所以空间复杂度为O(1)。

尽管这个解法在复杂度上满足了要求，但是修改了原始数据，不过修改原始数据只是为了代码方便，比如将负数归零也是改成在计数的时候判断是否为负数，如果是则计入nums[0]当中。
