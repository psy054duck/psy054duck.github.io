179. Largest Number

Description

Given a list of non negative integers, arrange them such that they form the largest number.

For example, given [3, 30, 34, 5, 9], the largest formed number is 9534330.

Note: The result may be very large, so you need to return a string instead of an integer.

Solution

题目描述很简单，就是将一个数组中的数字进行组合得到其中最大的数字。

思路很简单，大致方向就是利用字符串进行字典序降序排序，然后将排序后的数组中的数字一次拼接，即可得到最大的数字，算法中只用到排序，因此复杂度为$nlog(n)$。

当然，上述只是大致思路，排序时其实不完全是按照字典顺序排列。当一个字符串是另一个字符串前缀时，其大小不能只按字典序来判定。例如['534', '53']，按照字典序降序结果应该是['534', '53']，这样得到的数字是53453，但是很显然最大的数字应该是53534。因此，如果一个字符串是另一个字符串的前缀时，需要另一套规则进行大小判定。再举一个例子，['234', '23']，此时进行字典序降序则得到的数组时['234', '23']，这样得到的数字时23423，的确是最大的数。因此在一个字符串是另一个字符串前缀时有时也应按照字典序判定大小。

为了满足数字最大，需要将较大的数字放在高位，这样得到的数字才是尽可能的大。一个字符串是另一个前缀的情况下，这个公共前缀对形成的数字大小不会造成影响，因为两个字符串在这个部分都是一样的。因此需要考虑除去这个公共前缀后的情况。对于上述第一例，有两种排序的可能['534', '53'], ['53', '534']，这样两个结果的差别在于形成的数字百位一个是4，一个是5，可以发现4是’534’去除公共前缀’53’后留下的数字最高位，’5’是公共前缀’53’的最高位。所以原来两个数字的大小判定应该等价于['4', '53']，即较长的字符串除去公共前缀与较短的字符串的大小判定。注意，这个大小判定是可以递归的，例如['53534', '53']。
