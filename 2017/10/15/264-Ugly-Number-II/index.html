264. Ugly Number II

Description

Write a program to find the n-th ugly number.

Ugly numbers are positive numbers whose prime factors only include 2, 3, 5. For example, 1, 2, 3, 4, 5, 6, 8, 9, 10, 12 is the sequence of the first 10 ugly numbers.

Note that 1 is typically treated as an ugly number, and n does not exceed 1690.

Credits:
Special thanks to @jianchao.li.fighter for adding this problem and creating all test cases.

Solution 1

最简单的想法当然就是从1开始枚举，逐个检查是否为ugly number直至第n个出现。好想的代价就是效率低，所以这个方案我们不用。

Solution 2

ugly number的质因数只能是2, 3, 5，因此ugly number都符合公式$2^a3^b5^c$。显然，第k个ugly number一定前k-1个数中的某一个数乘以2, 3或5。于是我们可以使用三个指针a, b, c分别表示乘2, 3, 5数字的索引，并初始化为0。于是第k个ugly number一定是$min{nums[a]\times2, nums[b]\times3, nums[c]\times5}$，当a, b, c所代表的数字被用于获得第k个数之后，相应的指针加1。注意，nums用于存储已排序的前k-1个数。
