712. Minimum ASCII Delete Sum for Two Strings

发表于 2017-12-12 | 分类于 Leetcode |

Description

Given two strings s1, s2, find the lowest ASCII sum of deleted characters to make two strings equal.

Example 2:

Note:

0 < s1.length, s2.length <= 1000.

All elements of each string will have an ASCII value in [97, 122].

Solution

这题可以当作是最小编辑距离的改版。改动不大，首先是动作从3个（增、删、改）减少为1个（删），然后是将开销从动作的执行次数改为了删除字母的ascii之和。

这题做法就像教材里一样，依然是将字符串进行对齐，只是在评估开销的时候采用的是ascii值而已。所以状态转移方程可以写成以下形式：

​ $$s[i][j] = min(s[i-1][j-1]+diff(s1[i], s2[j]), s[i-1][j]+s1[i], s[i][j-1]+s2[j])$$

其中diff(s1[i], s2[j])的计算结果为s1[i] == s2[j] ? 0 : s1[i] + s2[j]。

同最小编辑距离一样，我们用s[i][j]表示针对s1[0…i]和s2[0…j]的最优解，于是这题的最终解就是s[s1.size()][s2.size()]。对于s1[i]与s2[j]而言，我们有三种选择，

  1. 将这两个字符对齐，于是开销由这两个字符是否相等来决定，如果相等那么这个对齐就不产生额外开销，于是开销就等于子问题s[i-1][j-1]，否则等于s[i-1][j-1] + diff(s1[i], s2[j])。
  2. s1[i]落空，不与任何字符对齐，那么开销就等于s[i-1][j] + s1[i]。
  3. s2[j]落空，不与任何字符对齐，那么开销就等于s[i][j-1] + s2[j]。

将三种情况的开销进行比较，取最小值，得到的就是s[i][j]。
